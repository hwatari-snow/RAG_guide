1. Search カラム拡張

Cortex Search Agent tool_resources

Cortex Search Service を Agent に登録する際、tool_resources で検索・メタデータ・ID・タイトルの各カラムを指定できます。

パラメータ 説明 本デモの設定例
search_columns 検索対象のテキストカラム（ベクトル検索+テキスト検索の対象） ["chunk_text"]
metadata_columns 検索結果に含めるメタデータカラム。Agent が回答生成時に参照可能 ["doc_title", "source_url", "file_name"]
id_column ドキュメントの一意識別子。自動引用（citation）の doc_id に使用される "source_url"
title_column ドキュメントのタイトル。自動引用の doc_title に使用される "doc_title"
max_results 検索結果の最大件数（デフォルト: サービス依存） 5
// Agent の tool_resources 設定例（SQL CREATE AGENT 内）
"tool_resources": {
  "search_caa_guidelines": {
    "search_service": "DEMO.KAIBALAB.CAA_GUIDELINE_SEARCH",
    "search_columns": ["chunk_text"],
    "metadata_columns": ["doc_title", "source_url", "file_name"],
    "id_column": "source_url",
    "title_column": "doc_title",
    "max_results": 5,
    "execution_environment": { "type": "warehouse", "warehouse": "COMPUTE_WH" }
  }
}
ポイント: id_column と title_column を設定すると、Agent フレームワークが自動的に citation（引用注釈）を生成します。LLM の指示文だけに頼るより正確なリンク出力が可能です。

2. 更新日時の優先（Time Decay）

Cortex Search

Cortex Search の Time Decay 機能を使うと、タイムスタンプカラムに基づいて新しいドキュメントを上位にランキングできます。

前提条件

  • Cortex Search Service のソーステーブルにタイムスタンプ型カラム（例: updated_at）が必要
  • CREATE CORTEX SEARCH SERVICE 時にそのカラムを ATTRIBUTES に含める

実装手順

Step 1: ソーステーブルにタイムスタンプカラムを追加

ALTER TABLE DEMO.KAIBALAB.RAG_PDF_CHUNKS
  ADD COLUMN updated_at TIMESTAMP_LTZ DEFAULT CURRENT_TIMESTAMP();

Step 2: Search Service を再作成（ATTRIBUTES に追加）

CREATE OR REPLACE CORTEX SEARCH SERVICE DEMO.KAIBALAB.CAA_GUIDELINE_SEARCH
  ON chunk_text
  ATTRIBUTES doc_title, source_url, file_name, updated_at
  WAREHOUSE = COMPUTE_WH
  TARGET_LAG = '1 hour'
AS (
  SELECT chunk_id, chunk_text, doc_title, source_url, file_name, updated_at
  FROM DEMO.KAIBALAB.RAG_PDF_CHUNKS
);

Step 3: クエリ時に scoring_config で Time Decay を指定

Agent が動的に time_decay パラメータを調整可能。または REST API / SQL から直接指定：

{
  "scoring_config": {
    "functions": {
      "time_decays": [{
        "column": "updated_at",
        "weight": 1,
        "limit_hours": 720  // 30日以上前のドキュメントはブーストなし
      }]
    }
  }
}
Agent との連携: Cortex Agent は検索パラメータ（filter、metadata_columns、結果数、time_decay 等）をユーザーのクエリに基づいて動的に調整できます。ツールの Column Description を適切に記述すると、Agent が自動的に最適なパラメータを選択します。

3. 参考リンク URL の自動引用（Citation）

Agent Cortex Search

Agent は Cortex Search の検索結果から自動的に引用注釈（citation annotation）を生成できます。

仕組み

id_column を設定した場合

検索結果の id_column の値が doc_id フィールドとして citation に含まれる。

例: "doc_id": "https://www.caa.go.jp/..."

title_column を設定した場合

検索結果の title_column の値が doc_title フィールドとして citation に含まれる。

例: "doc_title": "おとり広告に関する運用基準"

レスポンスの citation 構造

// Agent Streaming レスポンスの response.text.annotation イベント
{
  "type": "cortex_search_citation",
  "index": 0,
  "search_result_id": "cs_61987ff6-...",
  "doc_id": "https://www.caa.go.jp/policies/.../100121premiums_31.pdf",
  "doc_title": "おとり広告に関する表示等の運用基準",
  "text": "該当箇所のテキスト抜粋..."
}
ベストプラクティス: id_column に URL を設定すれば、クライアントアプリ側で doc_id をハイパーリンクとして直接使用できます。LLM に URL を推測させるよりはるかに正確です。

4. フィルター条件の活用

Cortex Search Agent 動的調整

Cortex Search の ATTRIBUTES カラムに対してフィルター条件を適用し、検索範囲を絞り込めます。Agent は Column Description に基づいてフィルターを動的に生成します。

// REST API での直接フィルター指定
{
  "query": "おとり広告の規制について",
  "filter": {
    "@eq": { "doc_title": "おとり広告に関する表示等の運用基準" }
  },
  "columns": ["chunk_text", "doc_title", "source_url"],
  "limit": 5
}

Agent での Column Description 設定

Snowsight UI の「Tool details」で各カラムの説明を記述すると、Agent が適切なフィルターを動的に生成します。

カラム Column Description 例
doc_title ガイドラインの正式名称。例: "おとり広告に関する表示等の運用基準", "不当景品類及び不当表示防止法の一部を改正する法律の施行について"
file_name PDFファイル名。例: "100121premiums_31.pdf"
source_url 消費者庁の公式ガイドラインPDFへのURL

5. スコアリングカスタマイズ

Cortex Search

検索結果のランキングを細かく制御できます。

機能 説明 ユースケース
Numeric Boosts 数値型カラム（likes, clicks 等）に基づくスコアブースト 人気度・閲覧数で上位表示
Time Decays タイムスタンプカラムに基づく新しさのブースト 最新のドキュメントを優先
Component Weights ベクトル検索・テキスト検索・リランカーの重み調整 キーワード重視 vs 意味重視のバランス
Reranking 無効化 リランカーを OFF にしてレイテンシを 100-300ms 削減 速度優先のリアルタイムアプリ
Scoring Profiles 名前付きスコアリング設定。複数クエリで再利用可能 本番環境での設定管理
Diversity 同一ドキュメントからの重複結果を制限 チャンクの偏り防止
// Diversity の例: 同一ファイルから最大2チャンクまで
{
  "scoring_config": {
    "diversity": {
      "group_by": ["file_name"],
      "max_results": 2
    }
  }
}

6. トークン数・推論時間の上限設定（Budget）

Agent Orchestration 重要

Agent の orchestration.budget で、1回のリクエストあたりのトークン上限と時間上限を設定できます。どちらかの上限に先に達した時点で Agent は応答生成を停止します。

パラメータ 型 説明 推奨値
seconds integer Agent のオーケストレーション処理の最大秒数 30〜120
tokens integer オーケストレーションに使用するトークンの最大数（Cortex Analyst / Search / ツール実行は含まない） 4000〜16000

SQL での設定（CREATE AGENT）

CREATE OR REPLACE AGENT DEMO.KAIBALAB.LEGAL_GUIDE_AGENT
  FROM SPECIFICATION $$
{
  "orchestration": {
    "budget": {
      "seconds": 60,
      "tokens": 8000
    }
  },
  ...
}
$$;

REST API での設定（agent:run リクエスト）

{
  "orchestration": {
    "budget": {
      "seconds": 30,
      "tokens": 16000
    }
  },
  "messages": [...]
}
注意: tokens パラメータはオーケストレーション（計画立案・リフレクション）にのみ適用されます。Cortex Analyst の SQL 生成や Cortex Search の検索で消費されるトークンは含まれません。全体のコスト制御にはリソースバジェット（次セクション）を併用してください。
Snowsight UI からの設定: Agent 編集画面 → Orchestration → Budget configuration で、時間制限とトークン制限をGUIから設定可能です。

7. リソースバジェット（月間クレジット上限）

Agent コスト管理

タグベースのリソースバジェットで、Agent オブジェクト単位の月間クレジット消費上限を設定できます。閾値到達時にストアドプロシージャを自動実行（通知 / アクセス停止 等）。

-- 1. タグ作成 & Agent に適用
CREATE TAG cost_mgmt.tags.cost_center ALLOWED_VALUES 'rag-demo';
ALTER AGENT DEMO.KAIBALAB.LEGAL_GUIDE_AGENT
  SET TAG cost_mgmt.tags.cost_center = 'rag-demo';

-- 2. Budget 作成（Snowsight UI: Admin → Cost management → Budgets）
-- 月間上限: 1000 credits
-- 80% 到達: メール通知
-- 100% 到達: アクセス停止（SP 実行）
Budget 反映遅延: 通常最大8時間（低レイテンシオプション有効時: 2時間）。80%閾値で早めの通知を推奨。

8. カスタムツール（UDF / Stored Procedure）

Agent

Stored Procedure や UDF を Agent のツールとして登録し、独自のビジネスロジックを実行できます。

ユースケース 実装方法
外部API連携 External Access Integration + UDF で外部 REST API を呼び出し
計算ロジック UDF で景品規制の金額計算など独自ロジックを実装
データ更新 Stored Procedure で Agent 経由のデータ書き込み
メール送信 Stored Procedure + SYSTEM$SEND_EMAIL で通知
// tool_resources でのカスタムツール設定
"tools": [{
  "tool_spec": {
    "type": "generic",
    "name": "calculate_premium_limit",
    "description": "景品類の最高額を計算する",
    "input_schema": {
      "type": "object",
      "properties": {
        "transaction_amount": { "type": "number", "description": "取引価額（円）" }
      },
      "required": ["transaction_amount"]
    }
  }
}],
"tool_resources": {
  "calculate_premium_limit": {
    "type": "function",
    "identifier": "DEMO.KAIBALAB.CALC_PREMIUM_LIMIT",
    "execution_environment": { "type": "warehouse", "warehouse": "COMPUTE_WH" }
  }
}

9. Web 検索ツール

Agent

Agent にリアルタイムの Web 検索機能を追加できます（Brave Web Search API 使用）。Snowflake 側で ZDR（Zero Data Retention）が有効化済み。

  • 有効化: Snowsight → AI & ML → Agents → Settings → Web search toggle
  • 用途: 最新の法改正情報、ニュース記事との組み合わせ
  • 注意: クエリと結果はパブリックインターネットを経由

10. Data to Chart（可視化）

Agent

Agent がクエリ結果から自動的にチャート（Vega-Lite 仕様）を生成します。tools 配列に含めるだけで有効化。

Cortex Analyst と組み合わせると、SQL クエリ結果の可視化が可能。

Agent Run API
type 説明

required 必ず1つ以上のツールを使用

12. 指示文の最適化（Instructions）

Agent

Agent の指示文は3種類に分かれ、それぞれ異なる目的に使用されます。

指示種別 用途 例
orchestration ツール選択・計画立案の方針 "返金に関する質問にはSearchツールを使用"
response 回答のトーン・形式・ブランド "常に日本語で回答。簡潔かつ丁寧に"
system システムレベルの基本指示 "あなたは景品表示法の専門アシスタントです"
コスト面の注意: 指示文が長いほど入力トークン数が増加します。必要最低限の指示にし、具体例は1-2個にとどめましょう。

13. スレッド（会話コンテキスト管理）

Agent

Thread オブジェクトを使うと、サーバー側で会話コンテキストが保持されます。クライアント側で履歴を毎回送信する必要がなくなります。

// 1. スレッド作成
POST /api/v2/cortex/threads

// 2. スレッド付きでリクエスト
POST /api/v2/databases/DEMO/schemas/KAIBALAB/agents/LEGAL_GUIDE_AGENT:run
{
  "thread_id": "<thread_id>",
  "parent_message_id": 0,
  "messages": [{ "role": "user", "content": [...] }]
}

14. フィードバック収集

Agent

ユーザーからの評価（positive/negative）とテキストフィードバックを収集し、Agent の改善に活用できます。

POST /api/v2/databases/DEMO/schemas/KAIBALAB/agents/LEGAL_GUIDE_AGENT:feedback
{
  "request_id": "<request-id>",
  "positive": true,
  "feedback_message": "正確な参照リンク付きで分かりやすかった",
  "categories": ["accuracy", "citations"]
}
